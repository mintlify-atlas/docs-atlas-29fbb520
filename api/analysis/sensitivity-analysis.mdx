The `SensitivityAnalysis` class provides tools for conducting global sensitivity analysis on models using Sobol indices and other methods.

## Class: SensitivityAnalysis

A class for performing sensitivity analysis on 0D models to identify which parameters have the most impact on model outputs.

### run_sobol_sensitivity

```python
run_sobol_sensitivity(sa_options=None)
```

Run Sobol global sensitivity analysis.

<ParamField path="sa_options" type="dict" default="None">
  Optional sensitivity analysis options
</ParamField>

**Returns:** Saves results to disk including:
- First-order Sobol indices (S1)
- Total-effect Sobol indices (ST)
- Second-order Sobol indices (S2)
- Plots and visualizations

**Notes:**
- Requires ground truth data and params_for_id to be set
- Runs in parallel using MPI
- Only rank 0 saves results and generates plots

### choose_most_impactful_params_sobol

```python
choose_most_impactful_params_sobol(
    top_n=5, 
    index_type='ST', 
    criterion='max', 
    threshold=0.01, 
    use_threshold=False
)
```

Ranks and selects the most impactful parameters based on Sobol indices.

<ParamField path="top_n" type="int" default="5">
  Maximum number of parameters to return
</ParamField>

<ParamField path="index_type" type="str" default="'ST'">
  Which Sobol index to use: 'ST' (total-effect) or 'S1' (first-order)
</ParamField>

<ParamField path="criterion" type="str or callable" default="'max'">
  Ranking criterion: 'max', 'mean', or a custom function
</ParamField>

<ParamField path="threshold" type="float" default="0.01">
  Minimum score required for parameter selection
</ParamField>

<ParamField path="use_threshold" type="bool" default="False">
  Whether to filter parameters below the threshold
</ParamField>

**Returns:** `list` - List of parameter names ranked by impact (empty list on non-zero ranks)

**Notes:**
- Only returns results on MPI rank 0, None on other ranks
- Prints ranked parameters with their scores
- Returns empty list if no parameters meet criteria
