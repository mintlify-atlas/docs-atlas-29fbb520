## Overview

The `CVS0DCppGenerator` class generates C++ code from CellML files for 0D cardiovascular models. It supports advanced features including delayed differential equations, coupling to 1D models, and external variable annotations.

**Source:** `~/workspace/source/src/generators/CVSCppGenerator.py`

### Methods

#### annotate_cellml()

```python
annotate_cellml() -> None
```

Annotates the CellML model with RDF metadata for delayed variables, coupling variables, and external connections. This is required before C++ generation.

**Actions:**
- Assigns unique IDs to all CellML entities
- Identifies delayed differential equation variables
- Marks input/output variables for 1D-0D coupling
- Creates RDF annotations in Turtle format
- Saves annotated model as `{file_prefix}-with-ids.cellml`
- Saves annotations as `{file_prefix}-with-ids--annotations.ttl`

**Example:**

```python
generator.annotate_cellml()
# Creates: cvs_model-with-ids.cellml
#          cvs_model-with-ids--annotations.ttl
```

#### generate_cellml()

```python
generate_cellml(inp_data_dict) -> None
```

Generates CellML files using `CVS0DCellMLGenerator` before C++ generation.

<ParamField path="inp_data_dict" type="dict" required>
  Input data dictionary for CellML generation (see CVS0DCellMLGenerator)
</ParamField>

**Example:**

```python
inp_data = {
    'generated_models_subdir': './generated',
    'file_prefix': 'cvs_model',
    'DEBUG': False
}
generator.generate_cellml(inp_data)
```

#### generate_cpp()

```python
generate_cpp() -> None
```

Generates C++ code from the annotated CellML model using libCellML's C generator.

**Returns:** None (writes C++ files to disk)

**Generated Files:**
- `{output_cpp_file_name}.h` - Model header file
- `{output_cpp_file_name}.cpp` - Model implementation
- `main.cpp` - Main file (if `create_main=True`)

**Features:**
- Implements delayed differential equations
- Handles 1D-0D coupling with flow/pressure boundary conditions
- Supports external variable connections
- Includes solver configuration

**Example:**

```python
generator.annotate_cellml()
generator.generate_cpp()
# Creates: model0d.h, model0d.cpp (if coupled to 1D)
#     or: cvs_model.h, cvs_model.cpp (standalone)
```

#### set_annotated_model_file_path()

```python
set_annotated_model_file_path(annotated_model_file_path: str) -> None
```

Use a pre-existing annotated CellML file instead of generating annotations.

<ParamField path="annotated_model_file_path" type="str" required>
  Path to existing annotated model RDF file (.ttl)
</ParamField>

## Workflow

Typical usage workflow:

1. **Generate CellML** (optional if already generated)
   ```python
   generator.generate_cellml(inp_data_dict)
   ```

2. **Annotate Model**
   ```python
   generator.annotate_cellml()
   ```

3. **Generate C++ Code**
   ```python
   generator.generate_cpp()
   ```

## 1D-0D Coupling

When `couple_to_1d=True`, the generator:

- Creates `model0d.cpp` and `model0d.h` (fixed names for linking)
- Annotates variables for coupling to 1D finite volume vessels
- Supports flow and pressure boundary conditions
- Handles inlet/outlet connections
- Includes resistance control variables
- Supports volume sum variables for tracking total blood volume

**Connection Info Format:**

```python
conn_1d_0d_info = {
    '1': {
        'vess0d_idx': 0,  # Index in 0D model vessels_df
        'vess1d_idx': 0,  # Index in 1D model
        'port_volume_sum': 0  # Optional: 1 if volume sum port
    },
    # ... more connections
}
```

## Dependencies

**Required:**
- `libcellml` - For model parsing, annotation, and code generation
- `rdflib` - For RDF graph creation and annotation
- `numpy`, `pandas`, `re`, `json`

**Optional:**
- `utilities.libcellml_helper_funcs` - Helper functions for CellML operations
- `utilities.libcellml_utilities` - Utility functions for issue reporting

## Notes

- Requires LibCellML to be installed (exits if not available)
- Annotations use custom URIs for delayed and coupling variables
- The generator creates RDF annotations in Turtle format
- C++ generation uses libCellML's built-in C code generator
- When coupled to 1D, output file is always named `model0d.cpp/h`
- The class needs significant refactoring (noted in source comments)
