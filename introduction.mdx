## What is Circulatory Autogen?

Circulatory Autogen automates combining CellML modules into a system model, then calibrating the model to experimental or clinical data.

Common use cases include coupled biochemical cell models, neuron-cardiomyocyte models, and patient-specific 0D blood flow networks.

## Key Features

- **Automated CellML model generation** from modular vessel/component arrays
- **Multi-algorithm parameter identification** including Genetic Algorithm, CMA-ES, and Bayesian Optimization
- **MCMC-based uncertainty quantification** with emcee
- **Sobol sensitivity analysis** and identifiability analysis
- **MPI-parallel optimization** on HPC systems
- **Multi-target code generation** to CellML, C++, and Python
- **OpenCOR and Myokit solver integration**
- **Custom module framework** for extensibility

## What Can Circulatory Autogen Do?

### 1. Model Autogeneration

Generate a system model by combining CellML modules in a user-defined network arrangement. Define your vessel array and parameters, then automatically generate coupled models.

### 2. Parameter Identification

Calibrate model parameters to experimental or clinical data using advanced optimization algorithms:

- Genetic algorithms
- CMA-ES (Covariance Matrix Adaptation Evolution Strategy)
- Bayesian optimization
- MCMC for uncertainty quantification

### 3. Sensitivity and Identifiability Analysis

Evaluate parameter influence and uncertainty with:

- Sobol sensitivity analysis
- Identifiability analysis using Laplace approximation or profile likelihood

## Use Cases

Circulatory Autogen is designed for researchers and engineers working on:

- **Cardiovascular system modeling** - Build patient-specific 0D blood flow networks
- **Coupled cell models** - Combine biochemical and electrophysiological modules
- **Multi-scale modeling** - Connect cellular models to tissue and organ-level systems
- **Parameter estimation** - Calibrate models to clinical measurements
- **Uncertainty quantification** - Assess confidence in model predictions
