Get up and running with Circulatory Autogen by generating and simulating a simple 3-compartment cardiovascular model.

## Prerequisites

Before starting, ensure you have completed the [installation](/installation) steps.

## Quick walkthrough

<Steps>
  <Step title="Navigate to the project directory">
    ```bash
    cd /path/to/circulatory_autogen
    ```
  </Step>

  <Step title="Configure user inputs">
    The example 3-compartment model is pre-configured. Open `user_run_files/user_inputs.yaml` and verify:

    ```yaml
    file_prefix: 3compartment
    input_param_file: 3compartment_parameters.csv
    model_type: cellml_only
    ```

    The `resources/` directory already contains:
    - `3compartment_vessel_array.csv` - Model structure definition
    - `3compartment_parameters.csv` - Model parameters
  </Step>

  <Step title="Run autogeneration">
    From the project root:

    ```bash
    cd user_run_files
    ./run_autogeneration.sh
    ```

    This generates CellML files in `generated_models/3compartment/`:
    - `3compartment.cellml` - Main model file
    - `3compartment_units.cellml` - Units definitions
    - `3compartment_parameters.cellml` - Parameter values
    - `3compartment_modules.cellml` - Component modules
    - `3compartment_parameters.csv` - Parameter table
  </Step>

  <Step title="Open in OpenCOR">
    Launch OpenCOR and open the generated model:

    ```
    File > Open > generated_models/3compartment/3compartment.cellml
    ```
  </Step>

  <Step title="Configure simulation">
    In OpenCOR's Simulation panel:
    - **Starting point:** 0 seconds
    - **Ending point:** 2 seconds  
    - **Point interval:** 0.01 seconds
    - **Solver:** CVODE (default)
  </Step>

  <Step title="Run simulation">
    Click **Run** to simulate the model. The simulation runs for 2 seconds of cardiac cycles.
  </Step>

  <Step title="Plot results">
    In the OpenCOR plot panel, add these variables to visualize:
    - `aortic_root.v` - Aortic flow rate
    - `pvn.v` - Pulmonary vein flow rate  
    - `heart.Vol` - Heart volume

    You should see periodic cardiac waveforms.
  </Step>
</Steps>

## Understanding the 3-compartment model

This simple model represents the cardiovascular system with three main regions:

- **Heart** - Pumps blood with periodic contraction
- **Systemic circulation** - Arteries (`aortic_root`, `svc`, `ivc`) and systemic terminal resistance (`terminal_sys`)
- **Pulmonary circulation** - Pulmonary artery (`pa`), pulmonary vein (`pvn`), and pulmonary terminal resistance (`terminal_pul`)

The vessel array defines how these components connect:

```csv
name,BC_type,vessel_type,inp_vessels,out_vessels
heart,nn,heart_mn,"ivc svc pvn","aortic_root pa"
aortic_root,vp,arterial_simple,heart,svc
svc,vp,venous_simple,aortic_root,heart
ivc,vp,venous_simple,terminal_sys,heart
pa,vp,arterial_simple,heart,terminal_pul
pvn,vp,venous_simple,terminal_pul,heart
terminal_sys,pp,terminal,aortic_root,ivc
terminal_pul,pp,terminal,pa,pvn
```

## Alternative workflows

### Run from Python script

Instead of the bash script, use Python directly:

```bash
cd src/scripts
python script_generate_with_new_architecture.py
```

### Use Jupyter notebooks

Interactive tutorials are available:

```bash
cd tutorial/interactive
jupyter notebook generation_and_calibration.ipynb
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Missing OpenCOR Python shell">
    Ensure `user_run_files/opencor_pythonshell_path.sh` points to your OpenCOR installation:

    ```bash
    OPENCOR_PYTHON=/path/to/OpenCOR/bin/python
    ```
  </Accordion>

  <Accordion title="Module generation errors">
    Check that all module types in `vessel_array.csv` exist in:
    - `src/generators/resources/*_modules_config.json`
    - `module_config_user/*_modules_config.json`
  </Accordion>

  <Accordion title="Simulation fails in OpenCOR">
    Verify parameters file has no missing values. Run autogeneration again to populate missing parameters with defaults.
  </Accordion>
</AccordionGroup>
